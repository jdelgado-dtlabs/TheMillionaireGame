## üìÖ Documentation Update Policy

### When to Update Documents

**`START_HERE.md`**: Update at the end of each session
**`V1.0_RELEASE_STATUS.md`**: Update when features complete or status changes
**`PRE_V1.0_TESTING_CHECKLIST.md`**: Update as bugs are fixed
**Session logs**: Create new file for each development session
**Checkpoints**: Create when pausing work for extended period

### Document Lifecycle

1. **Active** (`/active/`) - Current work in progress
2. **Sessions** (`/sessions/`) - Historical record of work done
3. **Archive** (`/archive/`) - Completed phases and outdated plans
4. **Reference** (`/reference/`) - Permanent technical documentation

### Archiving Criteria

Archive a document when:
- Feature is 100% complete and stable
- Plan has been fully executed
- Document is superseded by newer version
- Information is historical context only
